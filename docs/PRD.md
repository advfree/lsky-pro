## 1. 产品目标

对 Lsky Pro（兰空图床）进行安全加固、性能优化、UI 美化及集成图片自动压缩功能，将其打造成安全可靠、高性能、体验现代化的私有/公共图片托管平台。

## 3. 需求池

### 3.1 P0（必须做）

#### 3.1.1 安全漏洞修复

| 编号 | 安全类别 | 问题描述 | 修复方案 |
|---|---|---|---|
| SEC-01 | 文件上传安全 | `ImageService.php` 中仅通过 `$file->getClientOriginalExtension()` 获取扩展名做类型校验，可被 MIME 类型伪造绕过。 | 新增 `finfo` / MIME 内容检测 + 严格白名单校验；禁用 PHP 代码执行目录；上传目录禁止脚本执行。 |
| SEC-02 | 文件上传安全 | `store()` 中 `$file->getClientOriginalName()` 直接用于路径命名，可能包含路径遍历字符。 | 过滤或移除文件名中的 `../`、`..\\`、空字节等危险字符。 |
| SEC-03 | XSS 防护 | `origin_name` 在 Blade 模板中直接输出，未经过 `{{ }}` 自动转义（需排查全量视图）。 | 确保所有用户输入输出使用 Blade 的 `{{ }}`（已转义）而非 `{!! !!}`。 |
| SEC-04 | XSS 防护 | 图片链接（URL）在 `links()` 属性中拼接 HTML/BBCode/Markdown，如 `origin_name` 含恶意脚本则可能产生 XSS。 | 对 `origin_name` 和文件名字段进行 HTML 实体编码。 |
| SEC-05 | CSRF | API 路由 `/v1/upload` 使用 `auth:sanctum` 中间件部分不包含 CSRF 保护（无状态 API 需通过 Token 认证）。 | 确保 API Token 认证正确实现；Web 路由已启用 CSRF 中间件，需确认未遗漏。 |
| SEC-06 | SQL 注入 | `Image::scopeFilter()` 和 `rateLimiter()` 中使用了变量插值 `like "%{$keyword}%"`，但 Laravel Eloquent 默认参数绑定可防注入 — 需确认无原生 RAW 查询。 | 审计所有查询，禁止 `DB::raw()` 和 `whereRaw()` 直接拼接用户输入。 |
| SEC-07 | 路径遍历 | `image.php` 路由 `{key}.{extension}` 中 `extension` 虽用 `where` 限制，但 Controller `output()` 中 `$image->pathname` 用于读取文件，需防路径穿越。 | 确保 `pathname` 字段在存储时已被清理。 |
| SEC-08 | 敏感信息泄露 | `config('app.debug')` 为 `true` 时，异常信息直接返回给客户端（`ImageController.php:50`、`Controller.php:134`）。 | 生产环境强制关闭 `APP_DEBUG`；或新增中间件拦截 debug 错误输出。 |
| SEC-09 | 敏感信息泄露 | 缩略图路径是 `public/thumbnails/{md5}.png` 直接可公开访问。 | 缩略图文件使用随机化命名或受控访问。 |
| SEC-10 | 会话安全 | 未配置安全 Cookie 选项（Secure、HttpOnly、SameSite）。 | 在 `config/session.php` 中配置 `secure`、`http_only`、`same_site`。 |
| SEC-11 | API 认证 | `api.php` 路由中 `/v1/upload` 未强制要求 `auth:sanctum` 中间件（允许匿名上传）。 | 如需限制 API 必须认证，新增中间件。 |
| SEC-12 | 权限控制 | `Image::deleting()` 中删除物理文件时未校验当前用户是否有权限删除。 | 审计所有删除操作，确保权限验证。 |

#### 3.1.2 图片自动压缩功能

参考 EasyImages2.0 的图片处理架构，在 Lsky Pro 现有的 `ImageService::store()` 上传流程中集成自动压缩。

| 编号 | 功能 | 描述 |
|---|---|---|
| CMP-01 | 压缩质量配置 | 在用户组配置中增加「上传自动压缩质量」设置（百分比，默认 80%），支持 1-100 区间。 |
| CMP-02 | 图片尺寸限制 | 上传时自动检测图片尺寸，超过阈值（如 4000px）则等比缩放至阈值。 |
| CMP-03 | 格式自动转换 | 自动将不支持 Web 预览的格式（PSD、TIFF、BMP）转换为 WebP/JPEG/PNG。 |
| CMP-04 | WebP 输出支持 | 新增「输出格式」选项，支持自动转换为 WebP 格式以大幅减小体积。 |
| CMP-05 | 压缩前/后体积对比 | 上传成功后返回原始大小和压缩后大小，展示压缩率。 |
| CMP-06 | 按用户组配置 | 不同用户组可配置不同的压缩策略（质量/尺寸/格式）。 |
| CMP-07 | 跳过 GIF/SVG | GIF 和 SVG 格式跳过压缩，保持原始动画和矢量特性。 |
| CMP-08 | 后台压缩开关 | 管理员可在全局设置中启用/禁用自动压缩功能，并设默认参数。 |

**实现位置**: 在 `ImageService::store()` 中第 155-168 行的图片处理逻辑处扩展：
- 现有逻辑：`InterventionImage::make($file)->save(...)` 处理质量和格式
- 增强后逻辑：先判断是否启用压缩 → 如果启用，则将质量/尺寸/格式配置应用到图片处理 → 生成压缩后的文件

#### 3.1.3 响应速度优化

| 编号 | 优化项 | 描述 |
|---|---|---|
| PERF-01 | 数据库索引 | 为 `images` 表增加复合索引：`(user_id, created_at)`、`(strategy_id, md5, sha1)`、`(key, extension)`。 |
| PERF-02 | 图片懒加载 | 图片列表和画廊页面使用 `loading="lazy"` 属性，对首屏外的图片延迟加载。 |
| PERF-03 | 缓存策略 | 图片输出（`Controller::output()`）已有的缓存机制基础上，增加缓存标签（Cache Tags）支持；缩略图也纳入缓存体系。 |
| PERF-04 | Redis 缓存 | 支持使用 Redis 作为缓存驱动（目前默认 file），减少磁盘 IO。 |
| PERF-05 | 配置缓存 | `Utils::config()` 目前使用 `Cache::rememberForever`，增加监听配置变更事件以主动刷新缓存。 |

| PERF-08 | 图片 CDN 支持 | 在策略配置中增加 CDN 域名字段，URL 生成时使用 CDN 域名。 |
| PERF-09 | 压缩图片输出 | 结合 CMP-04 WebP 输出，在请求时根据 `Accept` 头自动返回 WebP 格式。 |
| PERF-10 | Octane 优化 | 利用已有的 Laravel Octane（Swoole/RoadRunner）配置，优化常驻内存应用的热路径。 |

## 5. 技术方案

### 5.1 安全方案

```
用户请求 → 中间件栈（CSRF / Auth / RateLimit / Session Security）
      ↓
  输入校验层（MIME 检测 / 扩展名校验 / 路径清理 / 文件大小限制）
      ↓
  业务逻辑层（权限校验 / 用户组配置校验 / 上传频率限制）
      ↓
  存储层（存储路径安全 / 禁止脚本执行 / 文件指纹校验）
```

### 5.2 图片压缩流程

```
用户上传 → 文件类型校验（跳过 GIF/SVG/ICO）
      ↓
  尺寸检测 → 超限则等比缩放
      ↓
  质量压缩（按用户组配置的质量百分比）
      ↓
  格式转换（可选：WebP/JPEG/PNG，自动转换 PSD/TIFF/BMP）
      ↓
  生成压缩统计（原始大小 / 压缩后大小 / 压缩率）
      ↓
  继续原有流程（水印 → 存储 → 缩略图 → 响应）
```

### 5.3 主题方案

- 使用 Tailwind CSS `darkMode: 'class'` 策略
- 在 `<html>` 标签上切换 `class="dark"`
- 主题偏好存储在 `localStorage('theme')` 中
- 默认跟随系统偏好 `prefers-color-scheme`
- CSS 变量定义颜色系统，亮/暗各一套变量

### 5.4 缓存策略

| 层级 | 方案 |
|---|---|
| 应用配置 | Redis Cache，配置变更时主动刷新 |
| 图片输出 | Redis + 文件缓存，TTL 可配置 |
| 缩略图 | 首次生成后持久化到 `public/thumbnails/` |
| 浏览器 | 图片响应头 `Cache-Control: public, max-age=2628000, etag` |
| 数据库查询 | 复杂统计查询缓存 5 分钟 |

## 6. 待确认问题

| 编号 | 问题 | 建议 |
|---|---|---|
| Q-01 | 压缩后的图片是否保留 EXIF 信息？ | 建议默认去除 EXIF（减小体积），保留可配置选项。 |
| Q-02 | WebP 转换是否为默认开启？ | 建议默认关闭（兼容性考虑），管理员自行开启。 |
| Q-03 | 暗色主题是否默认跟随系统还是默认亮色？ | 建议默认亮色，首次访问检测 `prefers-color-scheme` 后自动适配。 |
| Q-04 | 是否需要对已有图片做批量压缩？ | 建议 P0 仅处理新上传图片，P1/P2 再加入批量压缩后台任务。 |
| Q-05 | Docker 部署是否包含 Redis 和数据库？ | 建议提供标准 `docker-compose.yml` 包含 Nginx + PHP-FPM + MySQL/PostgreSQL + Redis。 |
| Q-06 | 权限模型是否需要细化？当前仅 Admin/User/游客三级。 | 建议保持三级权限模型，P2 可考虑引入角色 (Role) 机制。 |
| Q-07 | 图片自动压缩是否应用于 API 上传？ | 建议 API 上传同样应用压缩，可通过请求参数 `?compress=false` 绕过。 |
